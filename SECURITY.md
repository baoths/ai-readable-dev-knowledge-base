# Security policy

## Supported content

This repository is primarily documentation and snippets.
Security issues may still exist (e.g., unsafe guidance, insecure snippets, or secret leakage).

## Reporting a vulnerability

Please do **not** report security issues via public issues with full details.

Preferred reporting channel:
- Use GitHub Security Advisories ("Report a vulnerability") if enabled for this repository.

If the private reporting channel is not available:
- Open a public issue with **minimal** information (no exploit details, no secrets).
- Ask maintainers to provide a private contact method for full disclosure.

## What to include

- A clear description of the issue and its impact
- The affected file path(s)
- Suggested fix (if you have one)

## Response expectations

This is a best-effort project.
Maintainers will respond when possible and may prioritize issues based on impact and effort.
